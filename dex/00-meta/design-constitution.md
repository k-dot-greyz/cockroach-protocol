# Design Constitution

**The rules that keep Cockroach Protocol from becoming another cringe productivity app.**

---

## Core Rules

### 1. Style must intensify identity, not obscure action.

**Good:**  
- Dark palette with accent glows  
- Clean typography hierarchy  
- Ceremonial achievement unlocks  
- Encrypted-cabal aesthetic  

**Bad:**  
- Constant parallax  
- Autoplay chaos in data views  
- Loud animations during form entry  
- Decoration that competes with reading  

---

### 2. Primary workflows must remain calmer than promotional surfaces.

**Loud zones (go dramatic):**  
- Landing page hero  
- Product title cards  
- Achievement unlocks  
- Rank transitions  
- Leaderboard prestige views  
- Major dashboard overview  
- Incident mode activation  

**Quiet zones (whisper):**  
- Budget inputs  
- Runway calculator  
- Supply checklist entry  
- Long-form reading  
- Settings  
- Notes/logging  
- Anything the user touches while stressed  

**The shell can scream. The workflow should whisper.**

---

### 3. Numbers and next actions always outrank decoration.

**Hierarchy:**  
1. Next action ("Complete objective")  
2. Key metric ("23 days runway")  
3. Context ("Last updated 3h ago")  
4. Decoration (glitch halo, badge glow)  

If the user can't see the number or the action, the design failed.

---

### 4. Glitch is a signal, not a background condition.

**Use glitch for:**  
- Transition states  
- Instability warnings  
- Unlock moments  
- Alert conditions  
- Identity sting (brand moments)  

**Do not use glitch for:**  
- Constant jitter  
- Endless ambient noise  
- Aggressive parallax  
- Autoplay chaos  

If everything glitches, nothing feels special.

---

### 5. Prestige moments may spike in drama; data-entry moments may not.

**Ceremonial moments:**  
- Achievement unlocks — full-screen modal, badge reveal, title card  
- Rank transitions — animated ascent, leaderboard shift  
- Domain mastery milestones — prestige badge, ceremonial copy  

**Operational moments:**  
- Entering a budget line item — clean input, no fanfare  
- Updating runway data — instant save, subtle confirm  
- Checking off objectives — quick checkmark, no confetti  

The system celebrates **capability**, not **task completion**.

---

### 6. The interface should feel collectible, but also operable at low energy.

**Neurodivergent-optimized means:**  
- High visual interest when exploring  
- Low friction when operating  
- No forced decisions or modal traps  
- Clear exits from every screen  
- Saved state by default (no "Are you sure?")  

**Test:**  
Can the user open HQ at 3am, check runway, log an expense, and close — all in under 30 seconds, with zero cognitive load?

If not, the design failed.

---

## Visual Language

### Palette

**Base:**  
- Dark slate background  
- Off-white or cool gray text  
- Muted borders  

**Accent:**  
- Teal/cyan for primary actions and prestige  
- Amber/orange for warnings and hazards  
- Red for critical alerts  

**No:**  
- Bright, cheerful colors  
- Saturated primary palette  
- Gradient overload  

---

### Typography

**Display font:**  
- Bold, tight tracking  
- Used for: titles, achievement names, rank labels  

**Content font:**  
- Clean sans-serif, readable at small sizes  
- Used for: body text, descriptions, logs  

**Mono font:**  
- Terminal-style  
- Used for: data, metrics, code references, IDs  

---

### Motion

**Allowed:**  
- Smooth transitions (200–300ms ease)  
- Subtle hover states  
- Unlock reveals (ceremonial only)  
- Glitch stings (brief, intentional)  

**Banned:**  
- Autoplay animations in data views  
- Constant parallax  
- Bouncing UI elements  
- Aggressive screen shake  

**Rule:** Motion must have a reason (state change, unlock, alert). No motion for decoration.

---

## Component Rules

### Achievements

**Visual treatment:**  
- Badge as insignia, seal, patch, shard, or file stamp  
- Title in display font  
- Subtitle in mono or restrained sans  
- Tiny atmospheric animation on unlock  
- Optional glitch halo for rare achievements  

**Copy style:**  
Use compact, sharp lines.

**Example:**  
**Bureaucracy Eater**  
Benefits mapped. Hardship scripts prepared. Administrative panic reduced.

**Three-Month Creature**  
Runway extended beyond immediate fragility.

This should feel like a **classified commendation**, not an Xbox popup.

---

### Leaderboards (Domain Ascent)

**Good style cues:**  
- Callsigns or pseudonyms  
- Rank emblems  
- Minimal but sexy row styling  
- Trend arrows and delta movement  
- Seasonal labels like "Q2 HARDENING RUN"  

**Avoid:**  
- Giant avatars everywhere  
- Confetti  
- Bright gamified ribbons  
- Excessive crown/medal clutter  
- Cheap mobile-app score energy  

**Think:** Encrypted cabal, not classroom sticker chart.

---

### Panels and Cards

**Structure:**  
- Clear header with icon/label  
- Key metric or status (large, bold)  
- Supporting context (small, muted)  
- Optional action button (bottom-right)  

**No:**  
- Cards within cards within cards  
- Excessive shadows and depth  
- Rounded corners everywhere (use sparingly)  

---

## Sound and Texture

**If you ever add audio:**

**Good:**  
- Soft terminal chirps  
- Subtle confirmation ticks  
- Low synthetic pulses  
- Tactile click energy  

**Bad:**  
- Arcade rewards  
- Loud achievement fanfares  
- Constant UI noise  
- Distracting ambient hums  

**Texture can also come from:**  
- Visual grain  
- CRT dust  
- Scanlines  
- Subtle compression artifacts  

But keep them optional or lightweight.

---

## Anti-Failure Notes

**Ways this style could go wrong:**

1. **Too much glow, not enough hierarchy** — Everything glows, nothing stands out.  
2. **Too much glitch, not enough legibility** — User can't read the interface.  
3. **Too much persona, not enough utility** — Looks cool, but doesn't help.  
4. **Too much atmosphere, not enough completion flow** — User gets lost.  
5. **Too much cool, not enough trust** — Feels like a tech demo, not a tool.  

**The double hit:**  
User should think: _"Damn, this looks sick."_  
And then immediately after: _"Oh nice, this is actually easy to use."_

That double hit is the goal.
