# Vision

**Cockroach Protocol** is a financial resilience system for personal sovereignty.

---

## What It Is

- A **bespoke config** for neurodivergent minds  
- A **capability scaffolding** system for hardening against fragility  
- A **prestige layer** for quiet operators, not classroom gamification  
- A **physical-digital hybrid** using QR/NFC dead drops for real-world objectives  

---

## What It Is Not

- Not a productivity tool  
- Not a budgeting app with cartoon vibes  
- Not optimized for neurotypical executive function defaults  
- Not a social network for flex culture  

---

## Core Thesis

Financial resilience is not linear budgeting.

It is:

1. **Capability unlocking** — scaffolding nodes from dormant → active → secured → hardened  
2. **Domain mastery** — per-domain prestige (Runway, Bureaucracy, Gear, etc.)  
3. **Hazard navigation** — acknowledging regressions (snakes) and leaps (ladders)  
4. **Physical verification** — dead-drop objectives for proof of capability in the real world  

---

## User Archetype

**The Cockroach Operator**

- Neurodivergent, low baseline energy, high systems capacity when calm  
- Experiences income fragility, but capable of strategic hardening  
- Values meticulous tracking without gamified noise  
- Wants prestige without performing for an audience  
- Prefers encrypted-cabal aesthetic over bright, cheerful UX  
- Responds well to: ceremonial unlocks, quiet leaderboards, tactical objectives  
- Responds poorly to: confetti, loud notifications, forced socialization  

---

## Design Principles

1. **Capability first, hardening second, prestige third.**  
   - Unlock the node  
   - Harden the node  
   - Climb Domain Ascent  

2. **Glitches are signals, not background conditions.**  
   - Use glitch for: transitions, unlocks, alerts, identity stings  
   - Do not use glitch for: constant ambient noise  

3. **Loud zones vs quiet zones.**  
   - Go loud: landing hero, achievement unlocks, rank transitions  
   - Go quiet: budget inputs, runway calculator, settings, logs  

4. **Style intensifies identity, does not obscure action.**  
   - Numbers and next actions always outrank decoration  
   - The interface should feel collectible but operable at low energy  

5. **Progression is not linear.**  
   - Ladders: disproportionate gains (fixed-rate refinance, second income lane)  
   - Hazards: regressions (surprise repair, client loss, rent increase)

## Success Metrics (Internal)

- User feels **calm** while using the system  
- User feels **capable** after completing objectives  
- User feels **prestigious** when climbing Domain Ascent  
- User operates the system **at low energy** without friction  
- User returns to HQ as a **base of operations**, not a burden  

---

## Anti-Metrics

- Daily active users (DAU) — wrong frame, creates anxiety  
- Viral growth — not the goal  
- Engagement time — more time in-app = system failure  
- Social sharing — prestige is quiet, not performative  

---

## Long-Term Vision

Cockroach Protocol becomes:

1. A **personal sovereignty OS** — not just financial resilience  
2. A **network of quiet operators** — Domain Ascent as prestige layer  
3. A **physical-digital protocol** — dead drops as proof-of-capability rituals  
4. An **open standard** — other domains (health, knowledge, logistics) can plug in  

Eventually: MIDI as universal comms/translation layer between domains.
