# Bureaucracy Domain

**Benefits, credentials, and administrative hardening.**

Bureaucracy hardening reduces fragility by securing identity, benefits, and institutional access.

---

## Domain Purpose

Bureaucracy nodes protect against:

1. **Identity loss** — passport expiration, ID invalidity  
2. **Benefits forfeiture** — missed deadlines, unclaimed credits  
3. **Administrative paralysis** — lost documents, unclear protocols  
4. **Institutional lockout** — expired credentials, closed accounts  

---

## Core Principle

**Benefits are leverage. Credentials are keys. Protocols are armor.**

Bureaucracy mastery means:

- You know what you're entitled to  
- You have the documents to prove it  
- You have scripts for navigating systems  
- You have backups for everything  

---

## Node Categories

### 1. Identity Nodes

- **Passport (valid 6+ months)** (Secured: valid, Hardened: + backup ID + scanned copies)  
- **National ID** (Secured: valid, Hardened: + replacement protocol)  
- **Residence permit** (if applicable) (Secured: valid, Hardened: + renewal reminder system)  
- **Birth certificate** (Secured: original, Hardened: + certified copies + digital backup)  

### 2. Benefits Nodes

- **Unemployment insurance** (Secured: registered, Hardened: + claims protocol documented)  
- **Health insurance** (Secured: active, Hardened: + backup coverage + appeal scripts)  
- **Tax credits** (Secured: claimed, Hardened: + recurring claim automation)  
- **Housing benefits** (if eligible) (Secured: applied, Hardened: + renewal scripts)  
- **Child benefits** (if applicable) (Secured: claimed, Hardened: + automation)  

### 3. Financial Credentials

- **Bank account (primary)** (Secured: active, Hardened: + backup bank + documented access)  
- **Bank account (backup)** (Secured: open, Hardened: + minimum balance maintained)  
- **Payment processor account** (Wise, Revolut, PayPal) (Secured: verified, Hardened: + backup)  
- **Credit score access** (Secured: able to check, Hardened: + monitoring enabled)  

### 4. Document Hardening

- **Document vault** (Secured: physical folder, Hardened: + encrypted digital backup)  
- **Credential index** (Secured: list of all IDs, Hardened: + expiration tracking)  
- **Emergency protocol** (Secured: written down, Hardened: + tested + shared with trusted contact)

## Objectives (Sample)

### Easy

1. **Check passport expiration date**  
   - Locate passport  
   - Note expiration date  
   - Add renewal reminder (6 months before expiration)  

2. **Register for online tax portal**  
   - Visit national tax authority website  
   - Create account  
   - Verify identity  

3. **List all active benefits**  
   - Unemployment insurance?  
   - Health insurance?  
   - Tax credits?  
   - Write them down  

### Medium

1. **Open backup bank account**  
   - Research secondary bank options  
   - Open account  
   - Transfer €100 as minimum balance  

2. **Create document vault**  
   - Buy fireproof folder or safe  
   - Organize: passport, ID, birth cert, contracts, insurance  
   - Store in secure location  

3. **Scan all critical documents**  
   - Passport, ID, birth certificate, insurance cards  
   - Upload to encrypted cloud storage (Proton Drive, etc.)  
   - Test access from backup device  

### Hard

1. **Map all unclaimed benefits**  
   - Research national benefits database  
   - Identify 3+ benefits you're eligible for but not claiming  
   - Apply for at least one  

2. **Renew passport 6+ months early**  
   - Schedule appointment  
   - Gather documents  
   - Complete renewal  
   - Secure new passport  

3. **Document emergency protocol**  
   - What to do if primary ID is lost?  
   - What to do if bank account is frozen?  
   - What to do if you need to leave the country in 48 hours?  
   - Write scripts, test access, share with trusted contact

## Dead Drops (High Priority)

**Bureaucracy is the _primary domain_ for physical dead drops.**

**Why?**  
Bureaucracy nodes often require visiting physical locations:

- Embassies  
- Consulates  
- Government offices  
- Banks  
- Social services centers  

**QR/NFC dead-drop examples:**

### Easy (Public Access)

- **QR code at a bank branch** → "Open backup account" objective  
- **NFC tag at a public library** → "Access national benefits database" objective  
- **QR code at a post office** → "Verify identity documents" objective  

### Medium (Semi-Restricted)

- **QR code at a consulate entrance** → "Renew passport" objective  
- **NFC tag at a government office kiosk** → "Register for online tax portal" objective  

### Hard (Restricted Access)

- **QR code inside an embassy** → "Secure backup citizenship documentation" objective  
- **NFC tag at a benefits office** → "Claim hardship grant" objective  

**Difficulty tiers** are based on:

1. **Accessibility** — How easy is it to reach the location?  
2. **Bureaucracy** — Do you need an appointment? ID check?  
3. **Social friction** — Does it require talking to a clerk? Waiting in line?
