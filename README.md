**zenOS financial resilience system**

A bespoke config for personal sovereignty: capability scaffolding, domain ascent, dead-drop ops, and neurodivergent-optimized calm.

### Three Engines

1. **Product** — IA, game systems, protocols, loops  
2. **Persona** — callsigns, Domain Ascent, prestige, circles

### Core Screens

- **HQ** — operator base, system overview  
- **Console** — risk detection, hazard tracking

- **Objectives** — active missions and unlocks

- **Ascent** — Domain Ascent leaderboards and progression  
- **Logs** — event history and campaign records  

---

## Game Systems

### Node States

1. **Dormant** — identified but not active  
2. **Active** — in progress  
3. **Secured** — baseline capability achieved  
4. **Hardened** — resilience layer added  
5. **Locked** _(optional)_ — maintained at hardened state  

### Progression Logic

**Capability first, hardening second, prestige third.**

- Unlock nodes by completing objectives  
- Secure nodes to establish baseline capability  
- Harden nodes to add resilience layer  
- Domain Ascent is per-domain prestige — not a single global leaderboard  

### Domains

- **Runway** — financial scaffolding, burn rates, reserves  
- **Bureaucracy** — benefits, credentials, admin hardening  
- **Gear** — tools, hardware, redundancy  
- **Income** — revenue streams, client portfolio  
- **Digital** — infosec, backups, identity  
- **Network** — relationships, access, circles  
- **Skills** — capabilities, certifications  
- **Recovery** — health, sleep, neurotype optimization

## Physical Layer

### Dead Drops

- **QR stickers** — city-placed objectives and node unlocks (standard + UV/glow variants)  
- **NFC tags** — gear and bureaucracy nodes with proof-of-capability triggers  
- **Difficulty tiers** — easy (coffee shops), medium (public transit), hard (embassies, consulates)

- Loud zones: landing hero, achievement unlocks, rank transitions
